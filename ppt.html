EmoStar

"Tracking is common. Insight is rare. EmoStar transforms emotional logs into meaningful AI-driven insights."

Problem Statement

While digital journaling and mood tracking applications are widely available, most platforms only store emotional entries without generating meaningful insights. Users lack AI-driven pattern recognition, trigger detection, and structured emotional growth tracking.

Literature Review

Existing apps such as Daylio, Reflectly, Journey, and Moodnotes provide mood logging and journaling features. However, they primarily focus on recording emotions rather than analyzing long-term emotional patterns, multimodal inputs, or AI-driven storytelling.

Research Gap / Innovation

There is no integrated platform that combines multimodal journaling, NLP-based emotion detection, trend visualization, and personalized emotional storytelling into one unified system.

Dataset / Input

In EmoStar, the primary input consists of user-generated emotional entries which can be:

  • Text-based emotional logs (journaling sentences)
  • Multimodal inputs: optional photos or keyword tags
  • Time-stamped entries to track emotion over sessions

These inputs serve as raw emotional data which the system processes using NLP techniques for sentiment and emotion detection.

Before analysis, text inputs are cleaned, tokenized, and vectorized to support accurate emotion classification and pattern extraction.

System Methodology

The system pipeline consists of text preprocessing, emotion detection using NLP techniques, time-based trend analysis, and visualization of emotional insights. The output includes structured summaries and graphical representations of emotional evolution.

Live Execution / Demo

EmoStar is implemented as a working prototype available on GitHub. Users can input emotional logs and observe AI-generated emotional insights and trend visualizations.

  • Emotion detection from text
  • Pattern recognition across entries
  • Visual emotional trend graphs
  • AI-generated emotional summaries
